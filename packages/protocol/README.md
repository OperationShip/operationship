This repository serves as the absolute **Source of Truth** and the "DNA" for the OperationShip ecosystem. It is a standalone, industrial-grade schema library that defines the shared language between the AI Brain (`operationship-api`) and the User Interface (`operationship-web`).

## đźŹ—ď¸Ź Architectural Philosophy
To prevent "hallucinations" and "sync-errors" in a multi-repo environment, this protocol utilizes **Zod** for runtime validation and **TypeScript** for compile-time safety.

1.  **Strict Validation**: The AI output from Gemini is validated at the edge before it enters the system state.
2.  **Type Inference**: TypeScript interfaces are never written manually. They are inferred directly from Zod schemas to ensure 100% synchronization between validation logic and type definitions.
3.  **Cross-Repo Consistency**: By package-linking this repo, both the API and Web repos use the exact same data contracts.

### 1. The AI Brain (`operationship-api`)
The API uses `OperationGraphSchema` to validate the raw JSON string returned by Gemini. If the AI misses a field like `schemaVersion` or `riskScore`, the validation fails at the source, preventing corrupted data from being saved.

### 2. The Command Center (`operationship-web`)
The Frontend imports the inferred types (e.g., `OperationNode`, `UnitEconomics`) to strictly type its Zustand stores and React Flow components. This ensures that UI elements like the "Risk Glow" or "Ghost Transparency" are always backed by verified data.

## đź”Ť Inch-by-Inch: Schema Breakdown

The core logic resides in `src/schemas.ts` and is exported through `src/index.ts`.

### 1. System Enums & Constants
Defines the rigid categories allowed in the system:
* **NodeStatus**: `active` (standard) or `ghost` (AI suggestion).
* **FeatureTier**: `free`, `pro`, or `enterprise` (used for Stripe gating).
* **NodeType**: A fixed list of 12 categories (e.g., `supplier`, `revenue`, `sustainability`) that map to specific SVG icons in the UI.

### 2. Analytical Layer (Unit Economics)
When the "Unit Economics" toggle is active in the UI, this schema is populated:
* **monthlyCost / monthlyRevenue**: Non-negative numbers for financial simulation.
* **riskScore**: A 0â€“10 integer that triggers red/green UI alerts.
* **efficiencyRating**: A 0â€“100 percentage for operational performance.

### 3. Intelligence Layer (Master Insights)
Contains high-tier strategic data generated by Gemini for the side-panel:
* `automationOpportunity`, `wasteMonetization`, `failureWarning`, `taxStrategy`.

### 4. Structural Layer (React Flow)
Standardized objects for the graph engine:
* **OperationNode**: Contains X/Y coordinates and business `data`.
* **OperationEdge**: Defines flow between nodes, including `flowType` (e.g., `capital`, `labor`) and `isBottleneck` flags.

### 5. Root Project (OperationGraph)
The master object for a saved project, including `schemaVersion` (for future migrations), `projectId`, `projectName`, and global `toggles`.

## đź› ď¸Ź Developer Workflow

### Updating the Protocol
When adding a new feature to OperationShip:
1.  **Modify `src/schemas.ts`**: Add the new Zod field (e.g., `carbonFootprint: z.number()`).
2.  **Auto-sync**: The `index.ts` will automatically infer the new TypeScript type.
3.  **Build**: Run `npm run build` to generate the `dist/` folder.
4.  **Implement**: Update the API (to send the data) and the Web repo (to display it).

### Commands
* `npm install`: Install core dependencies (`zod`, `typescript`).
* `npm run build`: Compiles the protocol for distribution.
* `npm run dev`: Recompiles on every file change.
